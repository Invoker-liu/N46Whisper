## 新增需求

### 需求：TTS 文本输入
系统必须支持用户通过以下方式提供待转换的文本内容：
- 从本地上传文本文件（.txt、.srt、.ass）
- 从 Google Drive 选择文本文件
- 复用现有文件选择器交互逻辑

#### 场景：用户上传本地文本文件
- **当** 用户点击上传按钮并选择本地 .txt 文件
- **那么** 系统读取文件内容并显示确认信息

#### 场景：用户从 Google Drive 选择文件
- **当** 用户通过文件树选择 Drive 中的 .srt 文件
- **那么** 系统加载文件内容并准备转换

---

### 需求：TTS 模型与语音配置
系统必须提供以下配置选项供用户选择：
- TTS 模型（如 VITS、Piper）
- 语音类型（男声/女声、语言、音色）
- 输出音频格式（.mp3、.wav）

#### 场景：用户选择中文女声配置
- **当** 用户选择模型为 VITS、语言为中文、音色为女声
- **那么** 系统加载对应的预训练模型

#### 场景：用户选择日语配置
- **当** 用户选择语言为日语
- **那么** 系统显示可用的日语语音选项

---

### 需求：TTS 音频生成
系统必须将输入文本转换为语音音频文件，并支持：
- 长文本自动分段处理
- 显示转换进度
- 利用 GPU 加速推理

#### 场景：短文本转换成功
- **当** 用户提交少于 500 字的文本
- **那么** 系统在 30 秒内完成转换并显示下载按钮

#### 场景：长文本分段处理
- **当** 用户提交超过 2000 字的文本
- **那么** 系统自动分段处理并显示进度条
- **且** 最终合并为完整音频文件

---

### 需求：TTS 音频导出
系统必须支持生成的音频文件导出：
- 自动下载到本地
- 保存到 Google Drive（可选）
- 支持 .mp3 和 .wav 格式

#### 场景：音频自动下载
- **当** TTS 转换完成
- **那么** 浏览器自动触发音频文件下载

#### 场景：保存到 Google Drive
- **当** 用户选择保存到 Drive
- **那么** 系统将音频保存到原文件同目录
